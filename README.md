# Kanban Board

Пет-проект — канбан-доска с drag-and-drop, ролевой системой, комментариями и управлением пользователями.

## Стек

| Категория | Технология |
|-----------|-----------|
| UI | React 19, TypeScript |
| Стейт | Redux Toolkit |
| Роутинг | React Router v7 |
| Формы | React Hook Form + Zod |
| DnD | @dnd-kit/core, @dnd-kit/sortable |
| Иконки | Lucide React |
| Сборка | Vite |
| Хранилище | localStorage (без бэкенда) |

## Запуск

```bash
npm install
npm run dev
```

Приложение откроется на `http://localhost:5173`.

```bash
npm run build    # продакшн сборка
npm run preview  # предпросмотр сборки
npm run lint     # проверка линтером
```

## Возможности

### Канбан-доска
- Три колонки по умолчанию: **To Do**, **In Progress**, **Done**
- Drag-and-drop карточек между колонками и внутри колонки
- Прогресс-бар, приоритет (Low / Medium / High), счётчик комментариев
- Аватары исполнителей на карточке (стопка до 3 штук + `+N`)

### Задачи
- Клик по карточке открывает модалку с редактированием
- Изменение заголовка, описания, приоритета, прогресса (слайдер)
- Удаление задачи кнопкой при наведении на карточку

### Комментарии
- Панель комментариев в правой части модалки задачи
- Отправка через `Enter`, перенос строки — `Shift+Enter`
- Счётчик на карточке обновляется реактивно

### Исполнители
- Назначение из зарегистрированных пользователей приложения
- **Admin** видит всех пользователей
- **User** может назначить только себя или пользователей с ролью admin
- Снять назначение: admin — любого, user — только себя

### Авторизация и роли
- Регистрация / вход по email + пароль
- Первый зарегистрированный пользователь автоматически получает роль **admin**
- Смена имени и цвета аватара в профиле
- Смена пароля с подтверждением текущего

### Управление пользователями (только admin)
Страница `/admin`, доступ через "Manage users" в меню профиля:
- Таблица всех пользователей с ролями
- Создание нового пользователя с выбором роли (User / Admin)
- Повышение / понижение роли любого пользователя (кроме себя)
- Удаление пользователя (кроме себя)

## Роли

| Действие | User | Admin |
|----------|------|-------|
| Просматривать и редактировать задачи | ✅ | ✅ |
| Назначать себя исполнителем | ✅ | ✅ |
| Назначать других исполнителями | ❌ | ✅ |
| Открывать `/admin` | ❌ | ✅ |
| Создавать/удалять пользователей | ❌ | ✅ |
| Менять роли пользователей | ❌ | ✅ |

## Важно про данные

Все данные хранятся в **localStorage** — бэкенда нет. При очистке localStorage (`localStorage.clear()` в консоли браузера) все пользователи и задачи сбрасываются.

Если у существующих пользователей нет поля `role` (после обновления с более старой версии) — нужно очистить localStorage и зарегистрироваться заново, либо добавить поле вручную через DevTools → Application → Local Storage → `kanban_users`.
